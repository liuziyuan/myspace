## 🐍 Python

### 📥 安装方式

#### uv（推荐）

```bash
# Homebrew 安装
brew install uv
```

### 🔄 更新方式

```bash
# Homebrew 更新
brew upgrade uv
```

### 🗑️ 卸载方式

```bash
# Homebrew 卸载
brew uninstall uv
```

### 💡 项目中使用

```bash
# 创建新项目
uv init my-project
cd my-project

# 创建虚拟环境
uv venv                        # 使用默认版本
uv venv --python 3.12          # 指定 Python 版本
uv venv -p 3.11                # 简写形式

# 安装依赖（两种方式）
uv pip install requests          # 传统方式
uv add requests                  # 推荐方式，自动更新 pyproject.toml

# 运行脚本
uv run python main.py

# Python 版本管理
uv python install 3.12          # 安装特定版本
uv python list                  # 查看可用版本
```

## 💚 Node.js

### 📥 安装方式

```bash
# 安装 Volta
brew install volta
```

### 🔄 更新方式

```bash
# 更新 Volta
brew upgrade volta

# 安装新版本 Node
volta install node@20

# 查看已安装的版本
volta list all
```

### 🗑️ 卸载方式

```bash
# Homebrew 卸载
brew uninstall volta
```

### 💡 项目中使用

```bash
# 安装 Node 版本
volta install node@20

# 项目中使用特定版本
cd my-project
volta pin node@18

# 安装包管理器
volta install npm@latest
volta install pnpm@latest
volta install yarn@latest

# 项目固定包管理器版本
volta pin yarn@1.22.0

# 运行脚本（自动使用项目指定版本）
npm run dev
```

## ☕ Java

### 📥 安装方式

```bash
# 官方脚本安装
curl -s "https://get.sdkman.io" | bash

# 重新加载 shell 配置
source "$HOME/.sdkman/bin/sdkman-init.sh"
```

### 🔄 更新方式

```bash
# 更新 SDKMAN 本身
sdk selfupdate

# 安装新版本 Java
sdk install java 21.0.1-tem

# 查看已安装的版本
sdk list java | grep installed
```

### 🗑️ 卸载方式

```bash
# 卸载特定 Java 版本
sdk uninstall java 21.0.1-tem

# 完全卸载 SDKMAN
rm -rf ~/.sdkman
# 并删除 shell 配置文件中的相关行
```

### 💡 项目中使用

```bash
# 查看可用的 Java 版本
sdk list java

# 安装特定版本（推荐 Temurin/Eclipse Adoptium）
sdk install java 21.0.1-tem     # Java 21 LTS
sdk install java 17.0.9-tem     # Java 17 LTS
sdk install java 11.0.21-tem    # Java 11 LTS

# 临时切换版本（当前终端会话）
sdk use java 17.0.9-tem

# 设置默认版本
sdk default java 21.0.1-tem

# 项目中使用特定版本（在项目根目录）
sdk use java 17.0.9-tem

# 查看当前使用的版本
sdk current java

# 安装其他工具（Gradle、Maven 等）
sdk install gradle 8.5
sdk install maven 3.9.6
```

### ⚙️ Maven 项目级配置

#### 📥 安装和版本管理

```bash
# 查看可用的 Maven 版本
sdk list maven

# 安装特定版本
sdk install maven 3.9.6

# 临时切换版本
sdk use maven 3.9.6

# 设置默认版本
sdk default maven 3.9.6

# 查看当前版本
sdk current maven
```

#### 📁 项目级 .mvn 目录配置

在项目根目录创建 `.mvn` 目录，团队共享 Maven 构建配置：

```
my-project/
├── .mvn/
│   ├── maven.config          # Maven 参数配置
│   └── jvm.config            # JVM 参数配置
├── pom.xml
└── src/
```

##### 🔧 maven.config - Maven 参数

常用配置示例：

```properties
# 并行构建（每个 CPU 核心一个线程）
-T 1C

# 批量模式，非交互式
-B

# 默认不跳过测试
-DskipTests=false

# 指定编码
-Dproject.build.sourceEncoding=UTF-8
```

##### ☕ jvm.config - JVM 参数

常用配置示例：

```properties
# 内存配置
-Xmx2048m
-Xms512m

# 元空间大小
-XX:MaxMetaspaceSize=512m

# 使用 G1 垃圾回收器
-XX:+UseG1GC
```

##### 📄 settings.xml - Maven 配置文件

SDKMAN 安装的 Maven 默认使用用户级配置文件：

```bash
# 用户级配置文件位置
~/.m2/settings.xml

# 如果不存在，创建目录和文件
mkdir -p ~/.m2
touch ~/.m2/settings.xml
```

基础模板：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<settings xmlns="http://maven.apache.org/SETTINGS/1.2.0"
          xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
          xsi:schemaLocation="http://maven.apache.org/SETTINGS/1.2.0 
                              https://maven.apache.org/xsd/settings-1.2.0.xsd">
  
  <!-- 本地仓库路径（可选） -->
  <!-- <localRepository>${user.home}/.m2/repository</localRepository> -->
  
  <!-- 镜像配置 -->
  <mirrors>
    <mirror>
      <id>aliyun</id>
      <mirrorOf>central</mirrorOf>
      <name>Aliyun Maven Mirror</name>
      <url>https://maven.aliyun.com/repository/public</url>
    </mirror>
  </mirrors>
  
</settings>
```

项目级 settings.xml（通过 maven.config 指定）：

```properties
# .mvn/maven.config 中指定项目级 settings.xml
-s ${project.basedir}/.mvn/settings.xml
```

然后创建 `.mvn/settings.xml` 文件，配置项目特定的仓库或镜像。

#### 💡 使用场景

```bash
# 克隆项目后，SDKMAN 自动识别版本（如果项目有 .sdkmanrc）
sdk env install

# 手动切换到项目需要的 Java 和 Maven 版本
sdk use java 17.0.9-tem
sdk use maven 3.9.6

# 构建（自动使用 .mvn 目录中的配置）
mvn clean install

# 临时覆盖 .mvn 配置
mvn clean install -DskipTests=true
```

#### 📌 .sdkmanrc 示例（可选）

项目根目录可创建 `.sdkmanrc` 文件，自动切换版本：

```properties
java=17.0.9-tem
maven=3.9.6
```

进入项目目录时，SDKMAN 会提示自动切换版本：

```bash
cd my-project
# SDKMAN 提示: Using java version 17.0.9-tem in this shell.
# SDKMAN 提示: Using maven version 3.9.6 in this shell.
```
